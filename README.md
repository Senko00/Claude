# VK + simkran.ru posting bot

Локальный бот для Симбирского кранового завода:

1. AI (GigaChat, Сбер, бесплатный лимит) сам придумывает тему и пишет новость.
2. AI-генератор картинок (Pollinations.ai, полностью бесплатно, без ключа)
   рисует к ней изображение.
3. Черновик падает в папку `drafts/pending/`.
4. Вы открываете локальную страничку (`review_app.py`), смотрите текст
   и картинку и одной кнопкой одобряете или отклоняете.
5. При одобрении бот публикует новость сразу в двух местах:
   - **VK**: сообщество [vk.ru/simkran](https://vk.ru/simkran) — пост с фото.
   - **simkran.ru**: элемент инфоблока «Новости» на Bitrix — через встроенный
     REST API (входящий вебхук), без прямых правок в базе данных, чтобы не
     сломать кэш/индекс сайта.

Весь стек бесплатный и работает без VPN из РФ: Python, Flask, `vk_api`,
GigaChat API (free tier), Pollinations.ai, Bitrix REST API (входит в
лицензию «Бизнес»).

## Установка

```bash
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env` по инструкции ниже.

## 1. Токен VK-сообщества

1. Зайдите в сообщество → **Управление** → **Работа с API** → **Ключи доступа**.
2. Создайте ключ, отметьте права **«Стена»** и **«Фотографии»**.
3. Скопируйте токен в `VK_TOKEN`.
4. `VK_GROUP_ID` — числовой ID сообщества (без минуса), виден там же или
   в ссылке `vk.ru/club<ID>` (если у сообщества короткое имя `simkran`,
   ID можно посмотреть через любой vk id resolver или в настройках сообщества).

## 2. Ключ GigaChat (Сбер, бесплатно, без VPN)

Google Gemini заблокирован для аккаунтов из РФ, поэтому текст генерирует
российский GigaChat — работает напрямую, без VPN.

1. Зайдите на https://developers.sber.ru/studio/workspaces и авторизуйтесь
   через Sber ID (обычный номер телефона, банковская карта не нужна).
2. Создайте проект **GigaChat API**, выберите тариф **GigaChat API Lite /
   для физлиц** — на нём есть бесплатный лимит токенов.
3. В настройках проекта скопируйте **«Ключ авторизации»** (это Client ID
   и Client Secret, уже склеенные и закодированные в base64) — вставьте
   как есть в `GIGACHAT_API_KEY`.
4. `GIGACHAT_SCOPE=GIGACHAT_API_PERS` (для физлиц) уже стоит по умолчанию.

**Про SSL-сертификат:** API Сбера использует сертификат от российского
удостоверяющего центра (НУЦ Минцифры), которому Python может не доверять
«из коробки» — тогда `generator.py` упадёт с ошибкой SSL. Если это
случилось:

1. Скачайте сертификат НУЦ Минцифры (актуальная ссылка есть на странице
   https://developers.sber.ru/help/gigachat-api/certificates).
2. Впишите путь к скачанному файлу в `.env`:
   ```
   GIGACHAT_CA_BUNDLE=/путь/к/russian_trusted_root_ca.pem
   ```
   Если файл в формате `.cer`, сначала сконвертируйте:
   `openssl x509 -inform der -in cert.cer -out cert.pem`.

## 3. Bitrix REST на simkran.ru

1. Если модуль REST API ещё не установлен — **Marketplace → Решения →
   найти «REST API»** (бесплатный модуль от Bitrix, входит в лицензию Бизнес)
   и установить.
2. **Marketplace → Разработчикам → Другое → Входящий вебхук**.
3. Выдайте права на модуль **iblock**.
4. Скопируйте выданный URL вида `https://simkran.ru/rest/1/xxxxxxxx/`
   в `BITRIX_WEBHOOK_URL`.
5. Найдите ID инфоблока «Новости»: **Контент → Инфоблоки**, откройте нужный
   раздел — ID виден в адресной строке/списке. Впишите в `BITRIX_IBLOCK_ID`.

> Почему вебхук, а не прямая запись в БД, хотя у вас есть полный доступ к
> базе: Bitrix держит поверх таблиц кэш, поисковый индекс и обработчики
> событий. Запись напрямую в MySQL их не обновит, и новость может не
> показаться на сайте или сломать поиск/sitemap. REST API делает то же самое,
> что панель администратора, поэтому безопасен.

## Ежедневная работа

**Генерация черновика** (можно вручную или по расписанию):

```bash
python generator.py
```

Поставьте на cron / Планировщик заданий Windows, например раз в день:

```
0 9 * * * cd /path/to/project && venv/bin/python generator.py
```

**Проверка и публикация:**

```bash
python review_app.py
```

Откройте http://localhost:5000 — там черновик с картинкой и текстом,
кнопки «✅ Одобрить и опубликовать» / «❌ Отклонить». Одобрение сразу
публикует пост в VK и создаёт новость на simkran.ru.

## Структура проекта

```
generator.py     — генерация черновика (GigaChat + Pollinations)
review_app.py    — веб-страница модерации (Flask, localhost)
publisher.py     — публикация одобренного черновика в VK + Bitrix
ai_client.py     — обёртка над GigaChat API и Pollinations.ai
vk_client.py     — публикация поста с фото в VK
bitrix_client.py — создание новости в Bitrix через REST
drafts/pending/  — черновики на проверке
drafts/published/— опубликованные (архив)
drafts/rejected/ — отклонённые
```

## Важно понимать

- AI придумывает темы и факты **с нуля** — он может выдумать несуществующий
  заказ или цифры. Поэтому обязательно читайте текст перед тем, как нажать
  «Одобрить», а не просто смотрите на картинку.
- Бесплатные лимиты GigaChat и Pollinations могут периодически меняться
  провайдерами — если генерация начнёт падать с ошибкой 429 или про
  исчерпанный лимит токенов, подождите или снизьте частоту запуска
  `generator.py`.
- Код не тестировался с реальными токенами (у меня нет доступа к вашим
  VK/Bitrix/GigaChat credentials) — после заполнения `.env` в первую очередь
  стоит прогнать один черновик от начала до конца и проверить, что пост и
  новость действительно появились.
