# VK + simkran.ru posting bot

Локальный бот для Симбирского кранового завода:

1. AI (YandexGPT — модель, которая лежит внутри Алисы, бесплатный грант)
   пишет новость на русском по выбранной теме (или по фактам, которые вы
   сами вписали в форму).
2. AI-генератор картинок (Pollinations.ai, полностью бесплатно, без ключа)
   рисует к ней изображение — картинку всегда можно заменить своей.
3. Черновик появляется прямо на той же веб-странице — не нужно ничего
   запускать отдельно.
4. Вы смотрите текст и картинку и одной кнопкой одобряете или отклоняете.
5. При одобрении бот публикует новость сразу в двух местах:
   - **simkran.ru**: элемент инфоблока «Новости» на Bitrix, с картинкой —
     через небольшой PHP-скрипт (`bitrix_endpoint.php`), который вы один раз
     загружаете на сайт; он использует внутреннее API Bitrix
     (`CIBlockElement::Add`), тот же механизм, что и добавление новости из
     админки, — кэш/поиск обновляются штатно, база данных напрямую не
     трогается.
   - **VK**: сообщество [vk.ru/simkran](https://vk.ru/simkran) — пост с
     превью-карточкой (ссылка на новость на simkran.ru, VK сам вытягивает
     из неё картинку и заголовок). Прямая загрузка фото в сам пост VK не
     поддерживается для токена сообщества, см. раздел ниже.

Весь стек бесплатный и работает без VPN из РФ: Python, Flask, `vk_api`,
YandexGPT (free tier), Pollinations.ai, PHP (уже есть на хостинге simkran.ru).

## Установка (один раз)

**Вариант А — просто дважды кликнуть `start.bat`** (Windows). Он сам создаст
окружение, поставит зависимости и откроет браузер. В первый раз он создаст
`.env` из шаблона и попросит его заполнить — после этого запустите `start.bat`
ещё раз.

**Вариант Б — вручную:**
```bash
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env` по инструкции ниже, затем `python review_app.py`.

## 1. Токен VK-сообщества

1. Зайдите в сообщество → **Управление** → **Работа с API** → **Ключи доступа**.
2. Создайте ключ, отметьте права **«Стена»** (и «Фотографии», если есть —
   на публикацию не влияет, но не помешает).
3. Скопируйте токен в `VK_TOKEN`.
4. `VK_GROUP_ID` — числовой ID сообщества (без минуса), виден там же или
   в ссылке `vk.ru/club<ID>` (если у сообщества короткое имя `simkran`,
   ID можно посмотреть через любой vk id resolver или в настройках сообщества).

> **Почему в VK не «настоящее» фото, а карточка-превью.** Загрузка фото
> в пост токеном сообщества запрещена самим VK (`photos.getWallUploadServer`
> недоступен для токена сообщества, ошибка `[27] Group authorization
> failed`). Формально это решается пользовательским токеном с правами
> `wall,photos,groups`, но современные VK ID-приложения (кабинет id.vk.com)
> такие права вообще не выдают — только доступ к личным данным для входа на
> сайты. Получить такой токен можно только через полноценное
> review-приложение VK для бизнеса. Поэтому вместо загрузки файла бот
> прикрепляет к посту ссылку на статью на simkran.ru — VK сам подтягивает
> из неё картинку и заголовок в виде карточки (это зависит от того, что
> Bitrix отдаёт в `og:image`/`og:title` на странице новости — если карточка
> будет пустой, значит на сайте не настроены Open Graph теги, скажите — я
> помогу их добавить).

## 2. Ключ YandexGPT (Yandex Cloud, бесплатный грант, без VPN)

Google Gemini заблокирован для аккаунтов из РФ, поэтому текст генерирует
YandexGPT — та же модель, что отвечает у Алисы. Работает напрямую, по
обычному SSL, без танцев с сертификатами (в отличие от GigaChat).

1. Зайдите на https://console.cloud.yandex.ru и зарегистрируйтесь.
   Yandex Cloud обычно просит привязать банковскую карту для подтверждения
   личности — это стандартная защита от ботов, при бесплатном гранте
   списаний не будет, пока вы сами не подключите платный тариф.
2. Создайте (или возьмите уже созданный по умолчанию) **каталог** (folder)
   — на странице каталога скопируйте его **ID**, вставьте в `YANDEX_FOLDER_ID`.
3. Слева в меню — **Сервисные аккаунты** → создать аккаунт → выдать роль
   **`ai.languageModels.user`**.
4. Открыть этот сервисный аккаунт → **API-ключи** → создать ключ → скопировать
   в `YANDEX_API_KEY`.
5. `YANDEX_MODEL=yandexgpt-lite` уже стоит по умолчанию — эта модель быстрее
   и укладывается в бесплатный лимит с запасом.

## 3. Bitrix-эндпоинт на simkran.ru

На этой инсталляции «1С-Битрикс: Управление сайтом» модуль «Веб-сервисы»
(REST API) установлен, но не отдаёт методы `iblock.*`. Поэтому вместо
вебхука используется прямой PHP-скрипт, который выполняется на самом
сервере сайта.

1. Откройте файл `bitrix_endpoint.php` из этого проекта.
2. Сгенерируйте случайный секретный ключ, например командой:
   ```bash
   python -c "import secrets; print(secrets.token_hex(32))"
   ```
3. Вставьте его вместо `ЗАМЕНИТЕ_НА_СВОЙ_СЛУЧАЙНЫЙ_КЛЮЧ` в файле (строка
   `const SECRET = "...";`).
4. Загрузите файл на сайт simkran.ru — через файловый менеджер в админке
   Bitrix (**Контент → Файлы и папки**, туда же, где лежит `bitrix` и другие
   папки сайта) или через FTP от RU-CENTER. Файл должен лежать в корне
   сайта, рядом с папкой `bitrix/` (путь получится
   `https://simkran.ru/bitrix_endpoint.php`).
5. В `.env`:
   - `BITRIX_ENDPOINT_URL=https://simkran.ru/bitrix_endpoint.php`
   - `BITRIX_ENDPOINT_SECRET=` — тот же ключ, что вписали в PHP-файл.
6. Найдите ID инфоблока «Новости»: **Контент → Инфоблоки**, откройте нужный
   раздел — ID виден в адресной строке/списке. Впишите в `BITRIX_IBLOCK_ID`.

> **При каждом обновлении `bitrix_endpoint.php`** (если я пришлю новую
> версию) его нужно заново загрузить на сайт тем же способом — GitHub на
> сам сайт ничего не отправляет автоматически, только Python-часть
> обновляется через скачивание проекта.

> **Важно про безопасность.** Любой, кто узнает URL скрипта и секретный
> ключ, сможет создавать новости на сайте. Держите `.env` и код скрипта
> в секрете, не публикуйте их.

## Ежедневная работа — теперь всё в браузере

**Windows:** дважды кликните `start.bat`. Откроется браузер на
http://localhost:5000 — там:

- форма **«Сгенерировать новую новость»** — выберите тему из списка (или
  оставьте «Случайная») и, если хотите, впишите реальные факты в поле ниже
  (например «отгрузили кран КБ-405 в Казань, заказчик — ООО «Стройград»»),
  чтобы AI не выдумывал, а описал именно это;
- под черновиком — кнопка **«Заменить картинку»**, если сгенерированная
  не понравилась: выберите свой файл и нажмите — заменится перед
  публикацией;
- **«✅ Одобрить и опубликовать»** — публикует сразу в VK и на simkran.ru;
- **«❌ Отклонить»** — просто убирает черновик.

Чтобы остановить — закройте окно PowerShell/чёрное окно, которое открыл
`start.bat`.

**Mac/Linux** (или если не хотите `.bat`): `python review_app.py` — тоже
сам откроет браузер.

## Структура проекта

```
start.bat            — запуск одним кликом (Windows): окружение + сервер + браузер
generator.py          — генерация черновика (YandexGPT + Pollinations), с темой/фактами
review_app.py         — веб-страница: генерация, модерация, загрузка своей картинки
publisher.py          — публикация одобренного черновика в Bitrix + VK
ai_client.py          — обёртка над YandexGPT API и Pollinations.ai, список тем
vk_client.py          — публикация поста в VK (текст + карточка-превью со ссылкой)
bitrix_client.py      — вызов bitrix_endpoint.php на simkran.ru
bitrix_endpoint.php   — скрипт для загрузки на сайт, создаёт новость через Bitrix API
drafts/pending/       — черновики на проверке
drafts/published/     — опубликованные (архив, используется и для «не повторять темы»)
drafts/rejected/      — отклонённые
```

## Важно понимать

- AI придумывает темы и факты, если вы не дали свои, — он может выдумать
  несуществующий заказ или цифры. Поэтому обязательно читайте текст перед
  тем, как нажать «Одобрить». Если тема требует точности — впишите факты
  в форму генерации, тогда AI будет использовать только их.
- Бесплатные лимиты YandexGPT и Pollinations могут периодически меняться
  провайдерами — если генерация начнёт падать с ошибкой 429 или про
  исчерпанный лимит токенов/грант, подождите или попробуйте позже.
- Код не тестировался с реальными токенами (у меня нет доступа к вашим
  VK/Bitrix/Yandex credentials) — после заполнения `.env` в первую очередь
  стоит прогнать один черновик от начала до конца и проверить, что пост и
  новость действительно появились.

## Идеи по улучшению генеративного контента

- **Список тем** (`ai_client.THEMES`) уже используется для разнообразия —
  можно отредактировать/дополнить под реальную жизнь завода (например
  «сертификация», «визит делегации», «юбилей предприятия»).
- **Факты в форме** — самый надёжный способ избежать выдумок: если у вас
  есть реальный инфоповод (отгрузка, заказ, событие), впишите 2-3 факта, и
  AI просто оформит их в текст, не досочиняя.
- **Память о прошлых темах** — бот уже подмешивает в промпт заголовки
  последних опубликованных новостей, чтобы не повторяться; при желании
  можно расширить окно (сейчас 8 последних) в `generator.py`.
- **Фирменный стиль** — если у завода есть брендбук/тон коммуникации
  (обращение к читателю, характерные фразы, любимые слова), можно вписать
  примеры прямо в `PROMPT_TEMPLATE` в `ai_client.py` — YandexGPT неплохо
  копирует стиль по образцам.
- **Своя база фактов** — можно завести текстовый файл с реальными данными
  о продукции/клиентах и подмешивать в промпт релевантный кусок, чтобы AI
  всегда отталкивался от реальности, а не придумывал — если интересно,
  могу такое сделать.
