# VK + simkran.ru posting bot

Локальный бот для Симбирского кранового завода:

1. AI (YandexGPT — модель, которая лежит внутри Алисы, бесплатный грант)
   сам придумывает тему и пишет новость.
2. AI-генератор картинок (Pollinations.ai, полностью бесплатно, без ключа)
   рисует к ней изображение.
3. Черновик падает в папку `drafts/pending/`.
4. Вы открываете локальную страничку (`review_app.py`), смотрите текст
   и картинку и одной кнопкой одобряете или отклоняете.
5. При одобрении бот публикует новость сразу в двух местах:
   - **VK**: сообщество [vk.ru/simkran](https://vk.ru/simkran) — текстовый пост
     (без фото — VK не разрешает загружать фото на стену токеном сообщества,
     только полноценным пользовательским, который для обычных аккаунтов
     недоступен без ручной модерации со стороны VK, см. раздел ниже).
   - **simkran.ru**: элемент инфоблока «Новости» на Bitrix, с картинкой — через
     небольшой PHP-скрипт (`bitrix_endpoint.php`), который вы один раз загружаете
     на сайт; он использует внутреннее API Bitrix (`CIBlockElement::Add`), тот же
     механизм, что и добавление новости из админки, — кэш/поиск обновляются
     штатно, база данных напрямую не трогается.

Весь стек бесплатный и работает без VPN из РФ: Python, Flask, `vk_api`,
YandexGPT (free tier), Pollinations.ai, PHP (уже есть на хостинге simkran.ru).

## Установка

```bash
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env` по инструкции ниже.

## 1. Токен VK-сообщества

1. Зайдите в сообщество → **Управление** → **Работа с API** → **Ключи доступа**.
2. Создайте ключ, отметьте права **«Стена»** (и «Фотографии», если есть —
   на публикацию текста не влияет, но не помешает).
3. Скопируйте токен в `VK_TOKEN`.
4. `VK_GROUP_ID` — числовой ID сообщества (без минуса), виден там же или
   в ссылке `vk.ru/club<ID>` (если у сообщества короткое имя `simkran`,
   ID можно посмотреть через любой vk id resolver или в настройках сообщества).

> **Почему пост в VK без фото.** Загрузка фото на стену community-токеном
> запрещена самим VK (`photos.getWallUploadServer` недоступен для токена
> сообщества, ошибка `[27] Group authorization failed`). Формально это
> решается пользовательским токеном с правами `wall,photos,groups`, но
> современные VK ID-приложения (кабинет id.vk.com) такие права вообще не
> выдают — там только доступ к личным данным для входа на сайты. Получить
> такой токен можно только через полноценное review-приложение VK для
> бизнеса, это не разовая настройка. Поэтому в боте фото уходит только на
> simkran.ru, а в VK — текст.

## 2. Ключ YandexGPT (Yandex Cloud, бесплатный грант, без VPN)

Google Gemini заблокирован для аккаунтов из РФ, поэтому текст генерирует
YandexGPT — та же модель, что отвечает у Алисы. Работает напрямую, по
обычному SSL, без танцев с сертификатами (в отличие от GigaChat).

1. Зайдите на https://console.cloud.yandex.ru и зарегистрируйтесь.
   Yandex Cloud обычно просит привязать банковскую карту для подтверждения
   личности — это стандартная защита от ботов, при бесплатном гранте
   списаний не будет, пока вы сами не подключите платный тариф.
2. Создайте (или возьмите уже созданный по умолчанию) **каталог** (folder)
   — на странице каталога скопируйте его **ID**, вставьте в `YANDEX_FOLDER_ID`.
3. Слева в меню — **Сервисные аккаунты** → создать аккаунт → выдать роль
   **`ai.languageModels.user`**.
4. Открыть этот сервисный аккаунт → **API-ключи** → создать ключ → скопировать
   в `YANDEX_API_KEY`.
5. `YANDEX_MODEL=yandexgpt-lite` уже стоит по умолчанию — эта модель быстрее
   и укладывается в бесплатный лимит с запасом.

## 3. Bitrix-эндпоинт на simkran.ru

На этой инсталляции «1С-Битрикс: Управление сайтом» модуль «Веб-сервисы»
(REST API) установлен, но не отдаёт методы `iblock.*` (это проверяется
запросом `https://simkran.ru/rest/1/<код вебхука>/methods` — в списке нет ни
одного `iblock.*`, только базовые типа `profile`). Поэтому вместо вебхука
используется прямой PHP-скрипт, который выполняется на самом сервере сайта.

1. Откройте файл `bitrix_endpoint.php` из этого проекта.
2. Сгенерируйте случайный секретный ключ, например командой:
   ```bash
   python -c "import secrets; print(secrets.token_hex(32))"
   ```
3. Вставьте его вместо `ЗАМЕНИТЕ_НА_СВОЙ_СЛУЧАЙНЫЙ_КЛЮЧ` в файле (строка
   `const SECRET = "...";`).
4. Загрузите файл на сайт simkran.ru — через файловый менеджер в админке
   Bitrix (**Контент → Файлы и папки**, туда же, где лежит `bitrix` и другие
   папки сайта) или через FTP от RU-CENTER, если удобнее. Файл должен лежать
   в корне сайта, рядом с папкой `bitrix/` (то есть путь получится вида
   `https://simkran.ru/bitrix_endpoint.php`).
5. В `.env`:
   - `BITRIX_ENDPOINT_URL=https://simkran.ru/bitrix_endpoint.php`
   - `BITRIX_ENDPOINT_SECRET=` — тот же ключ, что вписали в PHP-файл.
6. Найдите ID инфоблока «Новости»: **Контент → Инфоблоки**, откройте нужный
   раздел — ID виден в адресной строке/списке. Впишите в `BITRIX_IBLOCK_ID`.

> **Важно про безопасность.** Любой, кто узнает URL скрипта и секретный
> ключ, сможет создавать новости на сайте. Держите `.env` и код скрипта
> в секрете, не публикуйте их. Секретный ключ должен быть длинным и
> случайным (не «12345» и не слово) — команда в шаге 2 выше как раз это
> и делает.

## Ежедневная работа

**Генерация черновика** (можно вручную или по расписанию):

```bash
python generator.py
```

Поставьте на cron / Планировщик заданий Windows, например раз в день:

```
0 9 * * * cd /path/to/project && venv/bin/python generator.py
```

**Проверка и публикация:**

```bash
python review_app.py
```

Откройте http://localhost:5000 — там черновик с картинкой и текстом,
кнопки «✅ Одобрить и опубликовать» / «❌ Отклонить». Одобрение сразу
публикует пост в VK и создаёт новость на simkran.ru.

## Структура проекта

```
generator.py       — генерация черновика (YandexGPT + Pollinations)
review_app.py      — веб-страница модерации (Flask, localhost)
publisher.py       — публикация одобренного черновика в VK + Bitrix
ai_client.py       — обёртка над YandexGPT API и Pollinations.ai
vk_client.py       — публикация текстового поста в VK
bitrix_client.py   — вызов bitrix_endpoint.php на simkran.ru
bitrix_endpoint.php — скрипт для загрузки на сайт, создаёт новость через Bitrix API
drafts/pending/    — черновики на проверке
drafts/published/  — опубликованные (архив)
drafts/rejected/   — отклонённые
```

## Важно понимать

- AI придумывает темы и факты **с нуля** — он может выдумать несуществующий
  заказ или цифры. Поэтому обязательно читайте текст перед тем, как нажать
  «Одобрить», а не просто смотрите на картинку.
- Бесплатные лимиты YandexGPT и Pollinations могут периодически меняться
  провайдерами — если генерация начнёт падать с ошибкой 429 или про
  исчерпанный лимит токенов/грант, подождите или снизьте частоту запуска
  `generator.py`.
- Код не тестировался с реальными токенами (у меня нет доступа к вашим
  VK/Bitrix/Yandex credentials) — после заполнения `.env` в первую очередь
  стоит прогнать один черновик от начала до конца и проверить, что пост и
  новость действительно появились.
